# 06 参数校验

## 深入理解@Validated和@Valid

### @Valid 与 @Validated 的基本概念与来源

`@Valid` 来自 Java Bean Validation 规范（JSR-303 / JSR-380），包路径为：

- Java EE：`javax.validation.Valid`
- Jakarta EE 9+：`jakarta.validation.Valid`

`@Validated` 来自 Spring Framework，包路径为 `org.springframework.validation.annotation.Validated`，是对 `@Valid` 的增强封装。

### @Valid 与 @Validated 的核心区别

| 特性 | @Valid | @Validated |
|------|--------|------------|
| 来源 | JSR-303/380 标准 | Spring 扩展 |
| 分组校验 | 不支持 | 支持（通过 `groups` 属性） |
| 嵌套/级联校验 | 支持（标注在字段上） | 不支持 |
| 作用位置 | 方法参数、字段、方法返回值 | 类、方法参数 |
| 方法级别校验 | 不支持 | 支持（标注在类上，配合 AOP） |

### @Validated 的作用机制

标注在 Controller 方法参数上时，由 Spring MVC 的 `HandlerMethodArgumentResolver` 处理，触发 Bean Validation 对请求参数进行校验，失败时抛出 `MethodArgumentNotValidException`。

标注在类上时，Spring 通过 `MethodValidationPostProcessor` 创建 AOP 代理，拦截方法调用并对参数/返回值执行校验，失败时抛出 `ConstraintViolationException`。

```java
@Service
@Validated  // 开启该 Bean 的方法级别校验
public class UserService {

    // 方法执行前，Spring AOP 会自动校验参数
    public void createUser(@NotBlank String name, @Min(18) int age) {
        // ...
    }
}
```

### @Valid 的作用机制

`@Valid` 触发标准 Bean Validation，核心用途是**级联校验**：当对象中包含嵌套对象时，需在嵌套字段上加 `@Valid` 才能触发对嵌套对象内部字段的校验。

```java
public class OrderDTO {

    @NotNull
    @Valid  // 触发对 AddressDTO 内部字段的校验
    private AddressDTO address;
}

public class AddressDTO {

    @NotBlank
    private String city;

    @NotBlank
    private String street;
}
```

### 能否相互替代

在 Spring MVC Controller 的方法参数校验场景中，两者基本可以互换：

```java
// 以下两种写法效果相同（不涉及分组和级联）
public Result create(@Valid @RequestBody UserDTO dto) { ... }
public Result create(@Validated @RequestBody UserDTO dto) { ... }
```

不可替代的场景：

- 需要分组校验：必须用 `@Validated(Group.class)`
- 需要级联校验（嵌套对象）：必须用 `@Valid`
- 需要 Service 层方法校验：类上必须用 `@Validated`

### 如何在springboot中启用参数校验功能

添加依赖：

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-validation</artifactId>
</dependency>
```

在 Controller 中使用：

```java
@RestController
@Validated  // 开启路径变量、请求参数的方法级校验
@RequestMapping("/users")
public class UserController {

    @PostMapping
    public Result create(@Validated @RequestBody UserDTO dto) {
        return Result.ok();
    }

    @GetMapping("/{id}")
    public Result get(@PathVariable @Min(1) Long id) {
        return Result.ok();
    }
}
```

统一异常处理：

```java
@RestControllerAdvice
public class GlobalExceptionHandler {

    // 处理 @RequestBody 校验失败（MethodArgumentNotValidException）
    @ExceptionHandler(MethodArgumentNotValidException.class)
    public Result handleValidException(MethodArgumentNotValidException e) {
        String message = e.getBindingResult().getFieldErrors().stream()
            .map(FieldError::getDefaultMessage)
            .collect(Collectors.joining(", "));
        return Result.fail(message);
    }

    // 处理路径变量、请求参数校验失败（ConstraintViolationException）
    @ExceptionHandler(ConstraintViolationException.class)
    public Result handleConstraintException(ConstraintViolationException e) {
        String message = e.getConstraintViolations().stream()
            .map(ConstraintViolation::getMessage)
            .collect(Collectors.joining(", "));
        return Result.fail(message);
    }
}
```

### 最佳实践

- Controller 方法参数（`@RequestBody`）：使用 `@Validated`
- 嵌套对象级联校验：在嵌套字段上使用 `@Valid`
- Service 层方法参数校验：类上加 `@Validated`，参数上加具体约束注解
- 分组校验：定义 `interface Create {}` / `interface Update {}` 分组，配合 `@Validated(Create.class)` 使用
- 统一在 `@RestControllerAdvice` 中处理校验异常，返回统一格式

## 常见的参数校验注解有哪些

以下是 Java Bean Validation（JSR-303 / JSR-380）中**最常用的参数校验注解**及其含义说明，适用于 Spring Boot 项目中的 DTO、实体类等字段校验。所有注解均来自 `javax.validation.constraints`（Java EE）或 `jakarta.validation.constraints`（Jakarta EE 9+）。

---

### 1. **空值校验**

#### `@Null`

- 要求被注解的元素 **必须为 `null`**。
- 常用于“禁用”某些字段。

#### `@NotNull`

- 要求被注解的元素 **不能为 `null`**。
- ⚠️ 注意：**允许空字符串 `""`**（对 `String` 类型而言）。

#### `@NotEmpty`

- 要求被注解的元素 **非 null 且长度/大小 > 0**。
- 适用于：
  - `String`：非 null 且 `trim()` 后长度 > 0
  - `Collection` / `Map` / `Array`：非 null 且 `size() > 0`

#### `@NotBlank`

- **仅用于 `CharSequence`（如 `String`）**
- 要求非 null 且 **去除首尾空白后长度 > 0**。
- 比 `@NotEmpty` 更严格（`"   "` 会被拒绝）。

> ✅ 推荐：字符串校验优先用 `@NotBlank`，集合用 `@NotEmpty`。

---

### 2. **长度与大小校验**

#### `@Size(min = x, max = y)`

- 校验元素的**长度或大小**。
- 适用于：
  - `String`：字符长度
  - `Collection` / `Map` / `Array`：元素个数
  - `Stream`（部分实现支持）
- 示例：`@Size(max = 50)` 表示最多 50 个字符或元素。

---

### 3. **数值范围校验**

#### `@Min(value = x)`

- 数值 **≥ x**（支持 `int`, `long`, `BigDecimal`, `BigInteger` 等）。

#### `@Max(value = x)`

- 数值 **≤ x**。

#### `@DecimalMin(value = "x")`

- 数值 **≥ x**，支持小数（字符串形式指定，避免精度丢失）。
- 可选 `inclusive = false` 表示 **> x**。

#### `@DecimalMax(value = "x")`

- 数值 **≤ x**，同样支持小数和 `inclusive` 控制。

#### `@Positive` / `@PositiveOrZero`

- `@Positive`：> 0  
- `@PositiveOrZero`：≥ 0  
- 适用于整数和小数类型。

#### `@Negative` / `@NegativeOrZero`

- `@Negative`：< 0  
- `@NegativeOrZero`：≤ 0  

---

### 4. **正则表达式校验**

#### `@Pattern(regexp = "正则表达式")`

- 要求字符串 **匹配指定正则表达式**。
- 示例：邮箱校验  

  ```java
  @Pattern(regexp = "^[\\w.-]+@[\\w.-]+\\.[a-zA-Z]{2,}$")
  private String email;
  ```

---

### 5. **日期时间校验**

#### `@Past`

- 时间必须是**过去**（不包含当前时刻）。

#### `@PastOrPresent`

- 时间必须是**过去或现在**。

#### `@Future`

- 时间必须是**将来**（不包含当前时刻）。

#### `@FutureOrPresent`

- 时间必须是**将来或现在**。

> 支持类型：`java.util.Date`, `java.time.LocalDate`, `LocalDateTime`, `Instant` 等。

---

### 6. **电子邮件校验**

#### `@Email`

- 简单校验是否符合电子邮件格式（基于正则）。
- 内部使用类似 `".+@.+\\..+"` 的宽松规则。
- 如需严格校验，建议配合 `@Pattern`。

---

### 7. **断言类校验**

#### `@AssertTrue`

- 要求被注解的 **boolean 方法或字段返回 `true`**。
- 常用于自定义逻辑校验：

  ```java
  @AssertTrue(message = "起始时间必须早于结束时间")
  public boolean isTimeRangeValid() {
      return startTime == null || endTime == null || startTime.isBefore(endTime);
  }
  ```

#### `@AssertFalse`

- 要求结果为 `false`（较少用）。

---

### 8. **数字格式校验（JSR-380 新增）**

#### `@Digits(integer = x, fraction = y)`

- 校验数值的**整数部分最多 x 位，小数部分最多 y 位**。
- 适用于 `BigDecimal`, `String`（表示数字）等。
- 示例：`@Digits(integer = 10, fraction = 2)` → 最多 10 位整数 + 2 位小数。

---

### 9. **其他实用注解**

#### `@URL`

- 校验字符串是否为合法 URL（基于 `java.net.URL` 解析）。

#### `@CreditCardNumber`（Hibernate Validator 扩展）

- 校验信用卡号（Luhn 算法），**非标准 JSR，需 Hibernate Validator**。

#### `@Length`（Hibernate Validator 扩展）

- 类似 `@Size`，但**仅用于字符串**，语义更清晰。

#### `@Range`（Hibernate Validator 扩展）

- 整合 `@Min` + `@Max`，支持 `min`/`max` 属性。

> ⚠️ 注意：带“Hibernate Validator 扩展”的注解**不属于 JSR 标准**，但 Spring Boot 默认集成 Hibernate Validator，可安全使用。

---

### 使用建议

- **基础校验**：优先使用标准 JSR 注解（`@NotBlank`, `@Size`, `@Email` 等）。
- **复杂逻辑**：结合 `@AssertTrue` 或自定义约束（`@Constraint`）。
- **分组校验**：配合 `groups` 属性实现不同场景（如创建/更新）的差异化校验。
- **嵌套对象**：别忘了在字段上加 `@Valid` 以触发级联校验。

这些注解配合 `@Valid` / `@Validated`，可构建强大而简洁的参数校验体系。

## 如何自定义校验注解

自定义校验注解需要两步：定义注解 + 实现校验逻辑。

### 定义注解

```java
@Documented
@Constraint(validatedBy = PhoneValidator.class)  // 指定校验器
@Target({ElementType.FIELD, ElementType.PARAMETER})
@Retention(RetentionPolicy.RUNTIME)
public @interface Phone {

    String message() default "手机号格式不正确";  // 默认错误信息

    Class<?>[] groups() default {};  // 分组校验（必须有）

    Class<? extends Payload>[] payload() default {};  // 负载（必须有）
}
```

### 实现校验器

```java
public class PhoneValidator implements ConstraintValidator<Phone, String> {

    private static final Pattern PHONE_PATTERN = Pattern.compile("^1[3-9]\\d{9}$");

    @Override
    public void initialize(Phone annotation) {
        // 可从注解中读取配置参数
    }

    @Override
    public boolean isValid(String value, ConstraintValidatorContext context) {
        // null 值由 @NotNull 处理，这里允许 null 通过
        if (value == null) {
            return true;
        }
        return PHONE_PATTERN.matcher(value).matches();
    }
}
```

### 使用自定义注解

```java
public class UserDTO {

    @NotBlank(message = "手机号不能为空")
    @Phone  // 使用自定义注解
    private String phone;
}
```

### 带参数的自定义注解

```java
@Documented
@Constraint(validatedBy = EnumValueValidator.class)
@Target({ElementType.FIELD, ElementType.PARAMETER})
@Retention(RetentionPolicy.RUNTIME)
public @interface EnumValue {

    String message() default "枚举值不合法";

    Class<?>[] groups() default {};

    Class<? extends Payload>[] payload() default {};

    int[] values();  // 自定义参数：允许的枚举值列表
}
```

校验器实现：

```java
public class EnumValueValidator implements ConstraintValidator<EnumValue, Integer> {

    private Set<Integer> allowedValues;

    @Override
    public void initialize(EnumValue annotation) {
        // 从注解中读取允许的枚举值
        allowedValues = Arrays.stream(annotation.values())
            .boxed()
            .collect(Collectors.toSet());
    }

    @Override
    public boolean isValid(Integer value, ConstraintValidatorContext context) {
        return value == null || allowedValues.contains(value);
    }
}
```

使用示例：

```java
public class ProductDTO {

    @EnumValue(values = {1, 2, 3}, message = "商品状态只能是 1、2、3")
    private Integer status;
}
```
